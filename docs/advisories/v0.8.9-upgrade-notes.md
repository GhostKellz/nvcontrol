## Runtime Fixes

### Overlay-Safe Vulkan Probe

`nvctl driver info` uses `vulkaninfo --summary` as optional evidence for 610+ Vulkan extension detection. On the validated local system, the raw helper could crash through `libMangoHud.so`. nvcontrol now launches the helper with overlay-disabling environment variables and clears explicit layer/preload variables before parsing output.

```mermaid
sequenceDiagram
    participant Nvctl as nvctl driver info
    participant Env as sanitized child environment
    participant Loader as Vulkan loader
    participant ICD as NVIDIA ICD

    Nvctl->>Env: set DISABLE_MANGOHUD=1 and DISABLE_VKBASALT=1
    Nvctl->>Env: clear VK_INSTANCE_LAYERS, VK_LAYER_PATH, LD_PRELOAD
    Env->>Loader: run vulkaninfo --summary
    Loader->>ICD: enumerate NVIDIA device and extensions
    ICD-->>Loader: extension list
    Loader-->>Nvctl: stdout
    Nvctl-->>Nvctl: parse known 610+ extension names
```

### Game Detection Migration

`sysinfo` 0.39 changed process refresh and process-name handling. nvcontrol now refreshes all processes through the current API and converts process names through the lossless path expected by the crate.

```mermaid
flowchart LR
    tick["auto-profile poll tick"] --> refresh["refresh_processes(ProcessesToUpdate::All, true)"]
    refresh --> processes["process table"]
    processes --> names["process name to_string_lossy"]
    names --> match["launcher/game profile matching"]
    match --> apply["optional delayed profile apply"]
```
